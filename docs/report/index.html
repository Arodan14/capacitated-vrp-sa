Vehicle Routing Problem Using Simulated Annealing

This report presents a capacitated Vehicle Routing Problem solution built in C++ using Simulated Annealing. The program reads the input instance from VRP(nodes).csv, constructs feasible routes under the required constraints, and exports machine-readable results (CSV/JSON) for charts and interactive maps in this report.

Problem Setting

200 Customers 1 Depot Capacity 100 SA Metaheuristic
  • Each route starts and ends at the depot.
  • Each customer is visited exactly once.
  • Vehicle capacity cannot exceed 100.
  • The objective is to minimize fixed and transportation costs.

Problem Overview

The Vehicle Routing Problem asks how a fleet of vehicles should leave a depot, visit all customers, satisfy their demand, and return to the depot at minimum cost. In this project, the cost combines a fixed truck cost and a load-based transportation cost.

Because VRP is computationally difficult, the project uses a heuristic approach instead of exact optimization. The goal is to reach a strong feasible solution in reasonable time.

Dataset Format

The solver reads the following CSV structure:

ID,X,Y,Demand
  • ID: node identifier
  • X, Y: node coordinates
  • Demand: customer demand
  • Node 0 is the depot

Algorithm Overview

Simulated Annealing starts from an initial feasible solution, explores neighboring solutions, and uses a temperature-controlled acceptance rule to avoid getting trapped too early in poor local minima.

  1. Load nodes and demands from the CSV file.
  2. Build a full distance matrix.
  3. Create an initial feasible solution using a greedy nearest-neighbor rule.
  4. Generate neighbor solutions using route swap and relocation moves.
  5. Accept better solutions directly and sometimes accept worse ones based on temperature.
  6. Keep track of the best solution found while the temperature cools down.
  7. Export annealing progress (CSV/JSON) and the final solution (JSON) for reporting.

Implemented Parameters

  • VEHICLE_CAPACITY = 100
  • NUM_CUSTOMERS = 200
  • ESTIMATED_VEHICLES_NEEDED = 33
  • COST_PER_TRUCK = 500.0
  • COST_PER_KM = 0.024
  • T_MAX = 100000
  • T_MIN = 1
  • ALPHA = 0.995
  • ITER = 1000

Complexity Summary

The greedy initial solution is approximately O(n^2).

The Simulated Annealing phase is approximately:

O(L * ITER * n)

where n is the number of customers, L is the number of cooling levels, and ITER is the number of iterations performed at each temperature.

Performance Interpretation

  • Higher exploration improved the final cost but increased runtime.
  • Lower cooling factors reduced execution time noticeably.
  • Distance alone did not fully explain solution quality because the cost function also includes load and truck cost.
  • This is why the report compares cost, distance, vehicles, and runtime together.

Discussion

  • Experiment A achieved the best cost, suggesting that the larger initial temperature improved exploration.
  • Experiment C finished much faster because the lower cooling factor reduced the number of temperature levels.
  • Experiments B and C produced the same final cost in this test, showing that faster convergence may reduce solution quality.
  • The results indicate a clear tradeoff between runtime and final solution quality.
  • Future work can include more parameter combinations, multiple random seeds, and stronger neighborhood operators such as 2-opt.

How to run this project

Source code and issue tracker: github.com/Arodan14/capacitated-vrp-sa (clone URL: https://github.com/Arodan14/capacitated-vrp-sa.git).

  1. Dataset: keep VRP(nodes).csv in the project root (the solver reads it from the working directory).
  2. Build (MSVC): open a developer shell (x64), cd into the repo, then compile with cl. Example (adjust the Visual Studio path if you use 2022 or another edition): 
cd path\to\VRP
cmd /c "\"C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Auxiliary\Build\vcvars64.bat\" && cl /EHsc /I include main.cpp src\Route.cpp src\Solution.cpp src\VRPProblem.cpp src\SimulatedAnnealingSolver.cpp src\SolutionVisualizer.cpp /Fe:vrp.exe"
  1. Run the solver: .\vrp.exe writes annealing_progress.csv, annealing_progress.json, and solution.json next to the executable. Use flags such as --prefix exp_a_ to prefix output filenames when comparing experiments.
  2. Report JSON in docs/report/assets/: copy or rename solver outputs for the bundled Experiment A filenames, or run python tools/generate_report_assets.py from the repo root when you already have exp_a_routes_graph.svg and progress CSV there.
  3. View this report: serve docs/report over HTTP (for example npx serve docs/report) so the browser can load JSON from assets/. Use Larger view on any chart or map to open a full-size dialog (route map redraws crisply; charts use a high-resolution canvas snapshot).
